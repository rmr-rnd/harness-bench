## Что такое harness_bench

harness_bench — фреймворк для автоматического сравнительного бенчмаркинга LLM-агентов. Он позволяет запускать одни и те же наборы задач на разных языковых моделях и агентных фреймворках, собирать результаты и сравнивать качество.

Типичный сценарий использования: ML-инженер хочет понять, насколько хорошо новая версия модели справляется с function calling, написанием кода и сложными агентными задачами — по сравнению с предыдущей версией или с эталонной моделью.

## Участники и внешние системы

| Участник | Тип | Роль |
|---------|-----|------|
| **ML-инженер / Исследователь** | Пользователь | Готовит YAML-конфиг, запускает прогоны, читает `summary.json` или Web UI, принимает решения по моделям |
| **LLM Provider** | Внешняя система | Языковая модель, которую тестируют. Любой OpenAI-совместимый endpoint: OpenAI, vLLM, OpenRouter, локальный сервер |
| **LLM Judge Provider** | Внешняя система | Модель-судья для оценки открытых ответов. Может быть той же моделью или другой (например, более сильной). Настраивается через `judge_model` в конфиге |
| **Tavily Search API** | Внешняя система | Веб-поиск для задач с флагом `web_search: true`. Опциональный — если ключ не задан, поиск недоступен |
| **BitGN / PAC1 API** | Внешняя система | Платформа для PAC1-бенчмарка. Предоставляет задачи через API и принимает результаты прогона. Используется только при включённом `pac1` бенчмарке |
| **Docker Hub / Registry** | Внешняя система | Реестр Docker-образов. Используется для загрузки образа агента (Hermes) и образов sandbox-окружений (SWE-bench, TheAgentCompany) |

## Границы системы

В контексте harness_bench находятся:
- Фреймворк (CLI, оркестратор, Web UI)
- Docker-контейнеры агентов и sandbox-окружений (управляются фреймворком)
- Хранилища результатов (файловая система, PostgreSQL)

За границей системы:
- Сами языковые модели
- Агентские фреймворки (Hermes, OpenClaw) — используются как Docker-образы
- Внешние API (Tavily, BitGN)
- Инфраструктура CI/CD, в которой может запускаться фреймворк
